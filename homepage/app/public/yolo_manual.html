이미지 객체 탐지: Image Object Detection (YOLO)

1. 분석 개요

이미지 객체 탐지(Object Detection) 기능은 업로드된 이미지 파일(JPG/PNG/WEBP/BMP 등)을 입력으로 받아, 이미지 내부의 객체를 자동으로 인식하고 Bounding Box(경계 상자)를 그려 결과를 제공합니다.

처리 결과는 각 이미지에 대해 (1) 박스가 그려진 결과 이미지와 (2) 탐지 결과 JSON 파일이 함께 생성됩니다.

본 분석 서비스는 Ultralytics YOLO 계열 모델을 기반으로 동작하며, 기본 모델은 다음과 같습니다: Ultralytics YOLO11

객체 탐지는 "어떤 물체가 어디에 있는지"를 찾는 작업입니다. 예를 들어 차량/사람/자전거 등을 찾아 이미지 위에 사각형 박스를 표시합니다.

2. 입력 파일(이미지) 형식

객체 탐지는 다음과 같은 이미지 파일 포맷을 입력으로 받습니다.

  • .jpg, .jpeg
  • .png
  • .webp
  • .bmp

3. 탐지 신뢰도 임계값(conf_thres)

객체 탐지 결과는 각 박스마다 confidence(신뢰도) 점수를 가지며, conf_thres는 “탐지 결과로 채택할 최소 신뢰도(0~1)”를 의미합니다.

설정 설명 권장 상황
conf_thres ↑ (높게) 더 확실한 객체만 남습니다.
노이즈(오탐지)가 줄어드는 대신, 작은/가려진 객체가 누락될 수 있습니다. 		정확도가 중요할 때, 오탐지 최소화
conf_thres ↓ (낮게) 더 많은 객체가 검출됩니다.
작은 객체나 애매한 객체가 잡힐 수 있지만, 오탐지 가능성도 증가합니다. 		놓치지 않는 것이 중요할 때, 후보를 넓게 보고 싶을 때
기본값 0.25 대부분의 일반적인 상황

4. 출력 결과(zip) 구조

분석 결과의 내부 구조는 아래와 같습니다.

  • images/: Bounding Box가 그려진 결과 이미지
  • json/: 이미지별 탐지 결과(JSON)

예시 구조:

  • yolo_results.zip
    • images/sample_001.jpg
    • images/sample_002.jpg
    • json/sample_001.json
    • json/sample_002.json

5. JSON 결과 형식

각 이미지에 대해 생성되는 JSON에는 이미지 정보와 탐지된 객체 목록이 포함됩니다.

5.1 JSON 필드 설명

필드 설명
image 원본 이미지 파일명
width, height 이미지 크기(px)
detections 탐지 결과 목록(객체 배열)
class_id 탐지 클래스 ID
class_name 탐지 클래스 이름(예: car, person 등)
confidence 탐지 신뢰도(0~1)
bbox_xyxy Bounding Box 좌표 [x1, y1, x2, y2] (픽셀 기준)

5.2 JSON 예시

아래는 JSON 결과 예시입니다. 

{
  "image": "sample_001.jpg",
  "width": 1280,
  "height": 720,
  "detections": [
    {
      "class_id": 2,
      "class_name": "car",
      "confidence": 0.8732,
      "bbox_xyxy": [120, 210, 430, 520]
    },
    {
      "class_id": 0,
      "class_name": "person",
      "confidence": 0.7614,
      "bbox_xyxy": [600, 180, 690, 540]
    }
  ]
}

6. 사용 방법 (MANAGER)

MANAGER에서 객체 탐지를 실행하는 일반적인 흐름은 다음과 같습니다.

  1. 이미지 선택: 여러 이미지를 한 번에 선택할 수 있습니다.
  2. conf_thres 설정: 기본값 0.25 권장, 필요 시 0~1 사이에서 조정합니다.
  3. 저장 경로 선택: 결과 zip가 저장되고, 자동으로 압축이 해제될 수 있습니다.
  4. 결과 확인: images/ 폴더에서 박스가 그려진 이미지를 확인하고, json/에서 수치 정보를 확인합니다.

탐지 결과가 너무 적게 나오면 conf_thres를 낮춰보세요(예: 0.25 → 0.15).
반대로 오탐지가 많으면 conf_thres를 올려보세요(예: 0.25 → 0.35).
