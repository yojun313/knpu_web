객체 탐지: Object Detection YOLO Grounding DINO

1. 분석 개요

객체 탐지(Object Detection) 기능은 업로드된 이미지/영상 파일을 입력으로 받아, 내부의 객체를 자동으로 인식하고 Bounding Box(경계 상자)를 그려 결과를 제공합니다.

처리 결과는 대상 파일에 대해 (1) 박스가 그려진 결과 파일과 (2) 탐지 결과 JSON 파일이 함께 생성됩니다.

본 분석 서비스는 다음 두 모델을 제공합니다:

  • YOLO (Ultralytics YOLO 계열) 일반 객체 탐지
    기본 모델: Ultralytics YOLO11
  • Grounding DINO 프롬프트(텍스트) 기반 탐지
    텍스트 프롬프트(예: person. red car. dog.)를 입력하면, 프롬프트와 관련된 객체를 찾아 박스를 표시합니다. 모델 정보: IDEA-Research/grounding-dino-base

YOLO는 “사람/차/자전거 등” 일반적인 객체를 자동으로 찾아줍니다.
Grounding DINO는 “프롬프트로 원하는 객체를 지정”해서 그 대상만 찾아줍니다.

2. 입력 파일 형식

2.1 이미지

객체 탐지는 다음과 같은 이미지 파일 포맷을 입력으로 받습니다.

  • .jpg, .jpeg
  • .png
  • .webp
  • .bmp

2.2 영상

영상 객체 탐지는 다음과 같은 비디오 파일 포맷을 입력으로 받습니다.

  • .mp4
  • .avi
  • .mov
  • .mkv
  • .webm

3. 탐지 신뢰도 임계값(conf_thres)

객체 탐지 결과는 각 박스마다 confidence/score(신뢰도) 점수를 가지며, conf_thres는 “탐지 결과로 채택할 최소 신뢰도(0~1)”를 의미합니다.

이 값은 YOLO(conf_thres)와 Grounding DINO(box_threshold)에 공통으로 적용됩니다. (UI에서는 conf_thres 하나만 입력받고, 내부에서 모델별 파라미터로 매핑됩니다.)

설정 설명 권장 상황
conf_thres ↑ (높게) 더 확실한 객체만 남습니다.
노이즈(오탐지)가 줄어드는 대신, 작은/가려진 객체가 누락될 수 있습니다. 		정확도가 중요할 때, 오탐지 최소화
conf_thres ↓ (낮게) 더 많은 객체가 검출됩니다.
작은 객체나 애매한 객체가 잡힐 수 있지만, 오탐지 가능성도 증가합니다. 		놓치지 않는 것이 중요할 때, 후보를 넓게 보고 싶을 때
기본값 0.25 대부분의 일반적인 상황

4. 프롬프트(prompt) 객체 탐지

Grounding DINO는 텍스트 프롬프트를 기반으로 객체를 탐지합니다. 프롬프트는 영문 소문자 + 마침표(.)로 구분하는 것을 권장합니다.

예시

  • person.
  • person. red car.
  • dog. bicycle.

프롬프트가 너무 일반적이면 오탐지가 늘 수 있으니, 가능한 한 구체적으로 작성하는 것을 권장합니다.

5. 출력 결과(zip) 구조

분석 결과의 내부 구조는 아래와 같습니다.

  • images/: Bounding Box가 그려진 결과 이미지(또는 프레임 캡처/결과 이미지)
  • videos/: Bounding Box가 적용된 결과 영상(영상 입력인 경우)
  • json/: 파일별 탐지 결과(JSON)

예시 구조:

  • detect_results.zip
    • images/sample_001.jpg
    • images/sample_002.jpg
    • videos/sample_video.mp4 (영상 입력 시)
    • json/sample_001.json
    • json/sample_002.json
    • json/sample_video.json (영상 입력 시)

6. JSON 결과 형식

각 파일에 대해 생성되는 JSON에는 파일 정보와 탐지된 객체 목록이 포함됩니다.

6.1 공통 필드(예시)

필드 설명
image / video 원본 파일명
width, height 해상도(px)
detections 탐지 결과 목록(객체 배열 또는 프레임별 배열)
label 또는 class_name 탐지 객체 이름(예: car, person 등)
confidence / score 탐지 신뢰도(0~1)
bbox_xyxy Bounding Box 좌표 [x1, y1, x2, y2] (픽셀 기준)

6.2 이미지(JSON) 예시

아래는 이미지 입력에 대한 JSON 결과 예시입니다.

{
  "image": "sample_001.jpg",
  "width": 1280,
  "height": 720,
  "detections": [
    {
      "class_id": 2,
      "class_name": "car",
      "confidence": 0.8732,
      "bbox_xyxy": [120, 210, 430, 520]
    },
    {
      "class_id": 0,
      "class_name": "person",
      "confidence": 0.7614,
      "bbox_xyxy": [600, 180, 690, 540]
    }
  ]
}

6.3 영상(JSON) 예시

아래는 영상 입력에 대한 JSON 결과 예시(프레임별 탐지)입니다.

[
  {
    "frame": 0,
    "detections": [
      {
        "label": "person",
        "score": 0.82,
        "bbox_xyxy": [120, 80, 220, 420]
      }
    ]
  },
  {
    "frame": 1,
    "detections": []
  }
]

7. 사용 방법 (MANAGER)

MANAGER에서 객체 탐지를 실행하는 일반적인 흐름은 다음과 같습니다.

  1. 미디어 타입 선택: image 또는 video를 선택합니다.
  2. 파일 선택: 여러 파일을 한 번에 선택할 수 있습니다.
  3. conf_thres 설정: 기본값 0.25 권장, 필요 시 0~1 사이에서 조정합니다.
  4. (선택) 프롬프트 포함: Grounding DINO 사용 시 프롬프트를 입력합니다.
  5. 저장 경로 선택: 결과 zip가 저장되고, 자동으로 압축이 해제될 수 있습니다.
  6. 결과 확인: images/, videos/, json/ 폴더에서 결과를 확인합니다.

탐지 결과가 너무 적게 나오면 conf_thres를 낮춰보세요(예: 0.25 → 0.15).
반대로 오탐지가 많으면 conf_thres를 올려보세요(예: 0.25 → 0.35).
